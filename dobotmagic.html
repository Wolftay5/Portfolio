Pick & Place : Dobot Magician Lite

Automatisation d'une tâche de manipulation industrielle et gestion des coordonnées spatiales d'un bras robotique articulé.

Changement de Repère et Optimisation du Code

L’objectif de ce projet était de programmer un bras articulé pour effectuer des cycles automatisés de déplacement de pièces à l'aide d'une ventouse pneumatique.

Le cœur du travail logiciel a consisté à manipuler sous Python des matrices de transformation homogène 4x4. Derrière ce nom technique se cachent simplement des tableaux de calculs mathématiques qui fusionnent la rotation et le déplacement dans l'espace. En m'appuyant sur la documentation du constructeur pour obtenir les dimensions exactes du bras, ces matrices ont permis de calculer précisément le changement de repère entre la base fixe du robot et son outil (l'extrémité de la ventouse) afin de viser juste.

Pour la trajectoire, nous utilisions le logiciel de simulation RoboDK pour générer le code machine finale. Cependant, dès que le cycle demandait un enchaînement de mouvements trop important, le logiciel refusait de générer le programme. Pour contourner ce blocage, j'ai utilisé RoboDK uniquement pour générer les blocs de mouvements simples dont j'avais besoin. J'ai ensuite récupéré ces morceaux de code pour les assembler, les structurer et les automatiser manuellement avec des boucles directement dans le fichier final. Cette méthode m'a permis de m'affranchir des limites du logiciel et de programmer autant de mouvements que nécessaire.

Démonstration du Cycle

La validation finale montre le bras robotique exécutant ses trajectoires de manière fluide et répétitive, activant la dépression de la ventouse pour saisir et déplacer les objets sans aucune collision.
